# ثروتي - منصة إدارة الحسابات المالية والديون الذكية 💼💰
### Tharwaty - Modern & Intelligent Financial Dashboard

منصة ويب متكاملة، احترافية وآمنة، مصممة لمساعدة الأفراد والشركات الناشئة على تتبع وإدارة شؤونهم المالية اليومية بكل سهولة وسلاسة. تدعم المنصة كشوفات المدخولات والمصروفات، وإدارة الديون والالتزامات مع سجل دفعات جزئية، وتحليلات ورسوم بيانية تفاعلية فورية.

---

## ✨ المميزات الرئيسية للمشروع (Key Features)

1. **لوحة تحكم ذكية تفاعلية (Interactive Dashboard):**
   - عرض فوري لإجمالي الرصيد الحالي، والتدفقات النقدية الواردة والصادرة.
   - مؤشر استهلاك الميزانية الشهرية مع تنبيهات ذكية عند الاقتراب من تجاوز الحد.
   - رسوم بيانية تفاعلية للمصروفات حسب التصنيف والمنحنى الشهري للتدفقات.

2. **كشف العمليات المالية (Transactions Tracking):**
   - إضافة، تعديل، وحذف العمليات المالية (مدخولات ومصروفات).
   - فلترة متقدمة حسب نوع العملية (مدخول/مصروف) وحسب التصنيفات المخصصة.
   - محرك بحث فوري وسريع للوصول للعمليات عبر الأوصاف والتصنيفات.

3. **إدارة الديون والالتزامات (Debt Management System):**
   - تتبع الديون المستحقة لك على الآخرين (ديون لي) أو الالتزامات المالية المتوجب سدادها (ديون عليّ).
   - تسجيل **دفعات السداد الجزئية** مع الملاحظات والتواريخ.
   - تحديث تلقائي لحالة الدين (مسدد بالكامل، مسدد جزئياً، غير مسدد) بناءً على مجموع الدفعات.
   - شريط تقدم بصري يوضح النسبة المئوية للمبلغ المسدد والمتبقي لكل دين.

4. **التقارير والإحصائيات المالية (Advanced Financial Reports):**
   - تحليلات بصرية مخصصة للتدفق المالي الشهري.
   - تصدير وطباعة التقارير المالية بكبسة زر واحدة بتنسيق طباعة احترافي (Print-friendly CSS).

5. **الملف الشخصي والإعدادات (Profile & Settings):**
   - تغيير العملة الافتراضية للتطبيق (SAR, USD, EUR, AED, KWD, QAR ... إلخ).
   - ضبط وتعديل قيمة الميزانية الشهرية المستهدفة.
   - تبديل المظهر بين **الوضع الليلي (Dark Mode)** و**الوضع النهاري (Light Mode)**.
   - تحديث بيانات الملف الشخصي وتغيير كلمة المرور بأمان.

6. **نظام حماية وتسجيل دخول متكامل (Secure Auth System):**
   - تسجيل دخول وإنشاء حساب آمن باستخدام البريد الإلكتروني وكلمة المرور.
   - دعم تسجيل الدخول الفوري عبر حساب جوجل (Google Sign-In).
   - حماية كاملة على مستوى الخادم وجلسات مستخدم مشفرة ومحمية.

## 🛠️ البنية البرمجية والتصميم النظيف (Clean Architecture)

تم بناء المشروع باتباع أفضل المعايير العالمية في تنظيم الملفات وفصل المسؤوليات:
- **Frontend:** React 19 + TypeScript + Vite + Tailwind CSS v4 + Framer Motion.
- **Backend API:** Vercel Serverless Functions (`/api/*`) لتوفير أداء فوري وقابلية للتوسع اللامحدود.
- **Database:** Supabase Client (PostgreSQL) مع استعلامات مرنة ومحمية بالكامل من محاولات Spam والاختراق.

```text
├── api/                   # خادم الواجهة الخلفية (Serverless API Routes)
│   ├── _supabase.js       # عميل قاعدة البيانات (مرونة عالية ضد التوقف)
│   ├── _wake.js           # آلية استعادة الخدمة التلقائية
│   ├── transactions.js    # إدارة العمليات المالية (CRUD)
│   ├── debts.js           # إدارة الديون والالتزامات (CRUD)
│   ├── debt-payments.js   # إدارة دفعات السداد الجزئية
│   ├── settings.js        # إعدادات الملف الشخصي والميزانية
│   └── stats.js           # محرك حساب الإحصائيات والتحليلات المالية
│
├── src/                   # الواجهة الأمامية (Frontend)
│   ├── components/        # المكونات المشتركة (Sidebar, Charts, etc.)
│   ├── pages/             # الصفحات الأساسية (Dashboard, Profile, Login, etc.)
│   ├── lib/               # عملاء الخدمات والاتصال بالـ API (supabase, apiFetch, googleAuth)
│   ├── App.tsx            # المكون الرئيسي وإعدادات التوجيه (Routing)
│   ├── main.tsx           # نقطة الانطلاق ومعالجة OAuth
│   └── index.css          # ملف الأنماط وتكوين Tailwind CSS v4
│
├── public/                # الملفات العامة والرموز (Favicons, SVGs)
└── package.json           # الحزم والاعتماديات البرمجية
```

## ⚙️ ملف إعدادات البيئة (Environment Variables)

للتشغيل المحلي للمشروع، يرجى إنشاء ملف `.env` في المجلد الرئيسي وإضافة المتغيرات التالية:

```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key

# إعدادات الدخول بواسطة جوجل (اختياري)
VITE_GOOGLE_CLIENT_ID=your_google_client_id
VITE_GOOGLE_AUTH_PROXY=your_google_auth_proxy
```

## 🚀 خطوات التشغيل المحلي (How to Run Locally)

1. **تثبيت الاعتماديات (Install Dependencies):**
   ```bash
   npm install
   ```

2. **تشغيل خادم التطوير (Run Dev Server):**
   ```bash
   npm run dev
   ```

3. **بناء المشروع للإنتاج (Build for Production):**
   ```bash
   npm run build
   ```

## 🔒 الحماية والتحقق من البيانات (Security & Validation)

- **التحقق من البيانات (Server-side Validation):** يتم التحقق من جميع الحقول المدخلة (المبالغ، التواريخ، الأسماء) على مستوى الـ API قبل إدراجها في قاعدة البيانات.
- **حماية جلسات المستخدم:** يتم تأمين كل اتصال بالخادم باستخدام رمز التحقق (JWT) المرسل كـ `Bearer Token` في ترويسة الطلب لضمان خصوصية البيانات وعزلها التام بين المستخدمين.
- **الوقاية من الـ SQL Injection:** نعتمد كلياً على عميل Supabase JS الآمن والمحمي ضد ثغرات الحقن والقرصنة.
